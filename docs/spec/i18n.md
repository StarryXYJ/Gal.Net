# i18n JSON 文件结构

主要分为编辑器 i18n 和运行时 i18n。
无头和非无头版本都是共用的。
启动器和无头启动器只包含运行时 i18n。
编辑器两者都有。

## 实际使用

条目中 `speaker` 和 `content` 参数存储 i18n key，运行时通过 `EntryContext.GetText()` 解析。

## 键结构与访问

DynamicLocalization 支持 **扁平**（`speaker.alice`）和 **树形**（`speaker → alice`）两种方式。存储时使用树形结构，读取用扁平结构方便操作。
同时支持动态多文件多源加载。

## 编辑器 i18n

`Editor` 为根节点，树形结构下面按页面分类，每个页面自己定义需要的 i18n 键。
编辑器 UI 通过 XAML 绑定直接引用 DynamicLocalization 的键，不显式调用代码。

```json
{
    "Editor": {
        "Settings": {
            "Title": "设置",
            "FontSize": "字体大小",
            "ApplyButton": "应用"
        }
    }
}
```

## 运行时 i18n

运行时 i18n 有两个根节点：
- **UI** —— 相对固定、变动较小的界面文本，按页面分类
- **Game** —— 游戏动态内容（说话人、选项、条目文本等）

### UI

UI 部分同样按页面分类，键路径与编辑器 XAML 绑定配合使用。

```json
{
    "UI": {
        "StartView": {
            "Start": "开始游戏",
            "Settings": "设置",
            "Exit": "退出"
        }
    }
}
```

### Game

Game 是游戏内容的 i18n，有专用翻译编辑页面（表格状，列=键名，行=语言，单元格=翻译）。

开发时按单语言优先开发，运行时通过键查找翻译文本。

#### 键生成规则

- 节点 ID：纯递增 int，删除不补空缺，键路径拼接为 `node_{id}`（如 `node_1`、`node_2`）
- 条目 ID：组内纯线性递增 int，键路径拼接为 `entry_{id}`（如 `entry_1`、`entry_2`）
- 说话人键：自动生成的递增唯一键（如 `speaker_1`、`speaker_2`）
- 文本条目的 content 固定使用键名 `Text`

完整键路径示例：
- `Game.Node.node_1.entry_5.Text` → 某条文本内容
- `Game.Speaker.speaker_1` → 说话人显示名
- `Game.Node.node_2.entry_1.Option1` → 选项文本

#### 接口定义

每个数据模型实现键生成接口，前缀由类内部决定：

```csharp
interface II18nKeyProvider
{
    string GetI18nKey(string suffix = "");
}
```

例如 GroupNode 生成 `node_{id}` 前缀，ComplexEntry 生成 `{entryId}` 前缀。

#### JSON 结构

```json
{
    "Game": {
        "Speaker": {
            "speaker_1": "艾丽丝",
            "speaker_2": "鲍勃"
        },
        "Node": {
            "node_1": {
                "entry_1": {
                    "Text": "你好，我是艾丽丝"
                },
                "entry_2": {
                    "Text": "你是谁？"
                }
            },
            "node_2": {
                "entry_1": {
                    "Option1": "接受",
                    "Option2": "不接受"
                }
            }
        }
    }
}
```

结构是固定的/预定义的，编辑器可自动从上下文推导出完整键路径，无需手动填写。

#### 加载优先级

开发者嵌入的 i18n JSON 作为默认资源。玩家可在游戏解压目录的 `localization/` 下放置同名 JSON 文件，**玩家文件覆盖开发者文件**，便于社区翻译。

开发者可导出各种语言的 JSON 文件供玩家下载参考。
